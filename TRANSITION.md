# Transition Guide

Stubborn Contract is the official continuation of Spring Cloud Contract, led by
[Marcin Grzejszczak](https://toomuchcoding.com), one of its original creators.

**Current status:** `0.1.0-SNAPSHOT` is live on Maven Central snapshots with `sh.stubborn`
coordinates, Spring Boot 4.x support, and a modular layout. See the
[Migration Guide](docs/migration-from-spring-cloud-contract.md) to switch.

- **What happened?** Spring Cloud Contract transitioned away from the Spring Cloud
  portfolio. This repository is its new home, led by the original creator.
- **What changed?** New Maven coordinates (`sh.stubborn` groupId), package rename from
  `org.springframework.cloud.contract` to `sh.stubborn.contract`, and a clean module
  split (core / spring / spring-boot / spring-cloud tiers).
- **What stays the same?** `spring.cloud.contract.*` Boot properties, YAML contract
  format, Groovy/Java/Kotlin DSL, and the Apache 2.0 license.
- **What should I do now?** Add the snapshot repo and import `stubborn-dependencies`
  BOM — the BOM pins compatible versions of Spring Cloud Stream, REST-Assured, and
  other required libraries so you only need one version declaration.
- **Backward compatibility:** Stubs generated by Spring Cloud Contract 5.x continue to
  work — the `"spring-cloud-contract"` WireMock matcher name is registered as an alias.

```xml
<repositories>
    <repository>
        <id>central-snapshots</id>
        <url>https://central.sonatype.com/repository/maven-snapshots/</url>
        <snapshots><enabled>true</enabled></snapshots>
        <releases><enabled>false</enabled></releases>
    </repository>
</repositories>

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>sh.stubborn</groupId>
            <artifactId>stubborn-dependencies</artifactId>
            <version>0.1.0-SNAPSHOT</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```
